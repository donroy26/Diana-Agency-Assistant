# Client Communication Rules

## Before Writing Anything

1. Read the agent's voice profile at `_shared/voice/[agent name].md`
   - If the file doesn't exist or Status is "NOT SET UP": produce a neutral professional draft and label it at the top: **[VOICE PROFILE NOT SET UP — humanize before sending]**
2. Read deal-file.md and the Session Log to understand the current context, stage, and what has already been communicated

## File Naming and Location

Every draft is its own file:
`_shared/cases/[CASE_ID]/comms/YYYY-MM-DD-[situation-slug].md`

Examples:
- `2026-05-16-post-showing-followup.md`
- `2026-05-16-inspection-response-buyer.md`
- `2026-05-16-competing-offer-notification.md`

## Comms Log

After writing each draft, append a one-line entry to the Comms Log table in deal-file.md:

| Date | File | Status |
|------|------|--------|
| 2026-05-16 | comms/2026-05-16-post-showing-followup.md | PENDING APPROVAL |

## Voice Application Rules

- Read the Sample Line in the voice profile as a calibration check. Does my draft sound like that?
- If the voice profile says "short sentences," write short sentences. If it says "formal," no contractions. Apply it literally.
- Never use language the agent would not use. If a phrase feels like generic real estate copy, replace it.
- Match the agent's sign-off style exactly as documented in the profile.

## Situations — Each Has Different Tone and Content

**First follow-up after showing:** Warm, low-pressure. Reference the specific property. Give the client space to respond. Don't ask them to make a decision.

**Competing offer notification to buyer:** Direct, calm, factual. No panic language. Explain the situation, lay out their options, give a deadline if there is one. Let the agent's tone do the reassuring.

**Inspection issue response (buyer):** Reassuring but honest. Acknowledge the finding, normalize it, outline what happens next. Don't minimize or catastrophize.

**Inspection issue response (seller):** Solution-focused. Acknowledge the finding, present options (repair credit, price adjustment, seller fixes), give the agent's recommendation. Avoid blame language.

**Financing delay update:** Calm, factual. Acknowledge the delay, state what's being done about it, give a realistic timeline. Avoid vague reassurance. Clients sense it.

**Pre-close anxiety check-in:** Warm, grounding. Remind them of the progress made. Give them the specific next steps. Keep it short. They're not reading long emails right now.

**Post-showing feedback request:** Brief, friendly. Ask what they thought. One or two questions, not a form. Leave room for honest feedback.

**Post-close thank you:** Personal, genuine. Reference something specific about the deal. Don't use a template closing line. It will feel like one.

## Never

- Present a draft as final or ready to send
- Use language the agent wouldn't use
- Leave the Comms Log entry blank
- Skip reading the voice profile
